# 知识图谱功能设计规范

> 本文档记录知识图谱功能的设计规范，包括关系定义、视觉设计和交互规范。

---

## 一、功能概述

知识图谱功能用于可视化展示知识之间的关联关系，支持用户直观地浏览知识网络结构。

### 1.1 核心特性

- **关系可视化**：解析知识内容中的链接关系，自动生成图谱
- **Obsidian 风格**：简洁圆形节点 + 灰色连线 + V形箭头
- **力导向布局**：节点自动分布，支持拖动
- **默认全显**：节点、连线、标签默认全部可见
- **悬浮/拖动高亮**：相关节点高亮，无关节点平滑淡出
- **可配置**：浮动设置面板，支持力导向参数调整

## 二、关系定义格式

### 2.1 格式规范

知识图谱支持两种链接格式：

| 格式 | 说明 | 示例 |
|------|------|------|
| `[[名称.md|关系]]` | WikiLink 格式，带关系名称 | `[[API设计.md|引用]]` |
| `[[名称.md]]` | WikiLink 格式，默认"关联"关系 | `[[配置说明.md]]` |
| `` `路径.md` `` | 反引号格式（@引用生成），自动识别关系 | `` `.claude/knowledges/API设计.md` `` |
| `` [[`路径.md`|关系]] `` | 混合格式，保留自定义关系 | `` [[`.claude/knowledges/API设计.md`|相似]] `` |

### 2.5 混合格式说明

当 WikiLink 格式内部包含反引号包裹的路径时（如 `` [[`.claude/knowledges/xxx.md`|关联关系]] ``）：
- 正确提取文件名（去除反引号和路径前缀）
- 保留用户指定的关系（如"关联关系"）
- 避免被反引号格式重复解析

### 2.2 支持的路径格式

**WikiLink 格式**：
- `[[知识A.md|关系]]` - 直接引用
- `[[./知识A.md|关系]]` - 相对路径
- `[[.claude/knowledges/知识A.md|关系]]` - 完整路径

**反引号格式**（@引用功能生成）：
- `` `知识A.md` `` - 直接引用
- `` `./知识A.md` `` - 相对路径
- `` `.claude/knowledges/知识A.md` `` - 完整路径
- `` `.claude/agents/智能体A.md` `` - 跨模块引用

### 2.3 关系识别规则

**WikiLink 格式**：
- 显式指定关系名称，如 `[[知识A.md|相似]]` 关系为"相似"
- 未指定则默认为"关联"

**反引号格式**：
- 根据路径自动识别关系类型：
  - `/agents/` → "引用智能体"
  - `/nodes/` → "引用节点"
  - `/workflows/` → "引用工作流"
  - `/commands/` → "引用命令"
  - `/resources/` → "引用资源"
  - `/abilities/` → "引用能力"
  - `/knowledges/` → "引用知识"
  - 其他 → "引用"

### 2.4 正则表达式

```typescript
// WikiLink 格式
const WIKI_LINK_REGEX = /\[\[([^\]|]+\.(?:md|mdx))(?:\|([^\]]+))?\]\]/g

// 反引号格式
const BACKTICK_LINK_REGEX = /`([^`]+\.(?:md|mdx))`/g
```

### 3.3 节点样式

**节点大小动态调整（基于出度）**：

节点大小在固定基础大小之上，根据出度（引用其他节点次数）动态调整：

```
计算公式：
sizeMultiplier = 1 + Math.sqrt(outDegree) * 0.3
finalNodeSize = baseNodeSize * Math.min(sizeMultiplier, 3)
```

**大小增长示例**：

| 出度 | 放大倍数 | 说明 |
|------|----------|------|
| 0 | 1.0x | 基础大小 |
| 1 | 1.3x | 引用其他节点1次 |
| 4 | 1.6x | 引用其他节点4次 |
| 9 | 1.9x | 引用其他节点9次 |
| 16 | 2.2x | 接近上限 |
| ≥25 | 3.0x（上限） | 最大放大倍数 |

**设计要点**：
- 使用平方根函数平滑增长，避免差异过大
- 最大放大倍数限制为 3 倍，保持视觉平衡
- 保留用户可配置的基础节点大小（`config.nodeSize`）
- 连线绘制同步适配节点大小变化

**默认颜色规则（基于出入度）**：

| 节点类型 | 出度 | 入度 | 颜色 | 说明 |
|----------|------|------|------|------|
| 活跃节点 | > 0 | ≥ 0 | `#2e86de` | 有出度的节点，深蓝色 |
| 汇节点 | = 0 | > 0 | `#9CA3AF` | 只有入度无出度，灰色（接收者） |
| 孤岛节点 | = 0 | = 0 | `#4B5563` | 无入度无出度，深灰色（未关联） |

**交互状态**（v5.3 优化）：

| 状态 | 颜色 | 透明度 | 说明 |
|------|------|--------|------|
| 焦点节点（悬浮） | `#4dabf7` | 100% | 亮蓝色，醒目跳跃 |
| 相关节点（相连） | 根据出入度 | 100% | 保持默认颜色，不再高亮 |
| 无关节点 | `rgba(200, 220, 240, 0.12)` | 12% | 极淡蓝灰，接近融入背景 |

**交互效果**（v5.3）：
- 悬浮节点：节点变大 0.5px，颜色统一变为亮蓝色
- 相关节点：保持默认颜色（根据出入度显示深蓝/灰色/深灰），不再高亮
- 无关节点：饱和度降至约 15%，亮度提升至约 95%，几乎融入背景

### 3.4 连线样式

| 状态 | 颜色 | 透明度 | 说明 |
|------|------|--------|------|
| 焦点连线（悬浮） | `#4dabf7` | 100% | 亮蓝色，与焦点节点一致 |
| 默认连线 | `rgba(156, 163, 175, 1)` | 100% | 灰色 |
| 无关连线 | `rgba(200, 220, 240, 0.25)` | 25% | 极淡蓝灰，几乎融入背景 |

**箭头**：V 形箭头，大小 = 连线粗细 × 4，颜色与连线一致
**关系标签**：字体大小可配置（默认 9px），背景透明度低，悬浮时蓝色边框和文字

**双向引用标签合并**：
当两个知识相互引用时（A→B 且 B→A），两条连线的关系标签合并为一个，换行显示：
```
A ══════════════════════→ B
        ┌──────────┐
        │ 引用      │  ← A→B 的关系（上方）
        │ 依赖      │  ← B→A 的关系（下方）
        └──────────┘
```
- 上行：正向关系（source.id < target.id）
- 下行：反向关系（source.id > target.id）
- 标签背景高度根据行数自动调整
- 标签宽度取两行中最宽的那一行
- 标签背景不透明，完全覆盖下面的连线

**悬浮状态标签显示**：
- 默认状态：显示所有关系标签，双向引用合并显示
- 悬浮节点时：只显示当前悬浮节点的**出度关系**（source 是当前节点）
- 效果：让用户专注于"当前节点影响哪些节点"

**淡蓝水墨质感**：
- 默认连线和箭头为灰色，悬浮时变为亮蓝色
- 非活跃元素极度褪色（接近白色），饱和度 10-15%
- 活跃元素保持鲜艳蓝色，饱和度 80-90%
- 活跃与非活跃对比度约 8:1，形成强烈焦点

### 3.5 标签样式

| 元素 | 规范 |
|------|------|
| 字体 | 400（非加粗） |
| 大小 | 配置值 × 0.6 |
| 颜色 | 相关：深灰 `#374151`，无关：中灰 `#6B7280` |
| 位置 | 节点下方，固定位置（不随悬浮变大而移动） |
| 间距 | 与节点边缘间距 2px |

### 3.6 渐变显示效果

**缩放级别与透明度**：
- 连线：始终可见（透明度固定为 1）
- 箭头和关系标签：`globalScale < 0.3` 完全透明，`globalScale 0.3 ~ 0.8` 渐变显示，`globalScale > 0.8` 完全显示
- 节点标签：`globalScale < 0.5` 完全透明，`globalScale 0.5 ~ 1.0` 渐变显示，`globalScale > 1.0` 完全显示
- 默认视图（zoomToFit 后约 0.8-1.0）：连线完全显示，箭头和关系标签完全显示，节点标签渐变显示

**悬浮/拖动效果**（v5.3）：
- 焦点节点（悬浮）：亮蓝色 #4dabf7，节点变大 0.5px
- 相关节点：保持默认颜色，透明度 = 1，不再高亮
- 相关连线：亮蓝色 #4dabf7，透明度 = 1
- 无关节点：极淡蓝灰，透明度 = 0.12
- 无关连线：极淡蓝灰，透明度 = 0.25

### 4.3 配置持久化

**存储位置优化（2026-03-03）：**

配置跟随项目存储,支持版本控制和团队协作：

- **Electron 环境**：`.ocean/knowledge-graph-config.json`（项目目录）
- **浏览器环境**：`localStorage` 的 `ocean-knowledge-graph-config` key
- 配置变更时自动保存,打开图谱时自动加载
- 不同项目可以有独立的图谱配置

**优势：**
1. 配置跟随项目,便于版本控制
2. 团队成员可以共享图谱配置
3. 不同项目可以有不同的图谱参数设置
4. 与业务数据存储保持一致（`.claude/` 目录同级存储配置）

### 5.1 入口交互

### 5.2 图谱交互

| 操作 | 效果 |
|------|------|

| 拖拽画布 | 平移视图 |

### 5.3 悬浮/拖动高亮效果（v5.3 优化）

```
悬浮或拖动节点 A 时：
- 节点 A：亮蓝色 #4dabf7，节点变大 0.5px（焦点）
- 节点 A 相连的节点：保持默认颜色（不高亮），100% 不透明（相关）
- 其他节点：极淡蓝灰 rgba(200, 220, 240, 0.12)，几乎融入背景（非活跃）
- 相关连线：亮蓝色 #4dabf7（焦点）
- 无关连线：极淡蓝灰 rgba(200, 220, 240, 0.25)，几乎融入背景
- 关系标签：仅相关连线显示，悬浮时蓝色边框和文字
- 节点标签：位置固定，相关节点深灰色，无关节点淡化
```

**配色要点**：
- 焦点节点：亮蓝 #4dabf7，100% 不透明
- 相关节点：保持默认颜色（深蓝/灰色/深灰根据出入度），100% 不透明，不再高亮
- 无关节点：极淡蓝灰，透明度 12%，饱和度 15%，亮度 95%
- 焦点连线：亮蓝 #4dabf7，100% 不透明
- 相关连线：亮蓝 #4dabf7，100% 不透明
- 无关连线：极淡蓝灰，透明度 25%
- 对比度：焦点与非活跃约 8:1，非活跃与背景约 1.2:1

**逻辑变更（v5.3）**：
- 悬浮节点时，相关节点（相连节点）不再高亮为亮蓝色，而是保持其默认颜色
- 相关节点也不再淡化，保持 100% 透明度
- 只有悬浮节点本身和无关节点会有视觉变化
- 相关连线仍保持高亮（亮蓝色），方便追踪关联关系

### 5.4 平滑过渡动画（v5.2）

**动画时长**：500ms

**缓动函数**：`easeOutCubic` - `1 - Math.pow(1 - progress, 3)`

**过渡内容**：

| 元素 | 过渡类型 | 说明 |
|------|----------|------|
| 无关节点透明度 | 1 ↔ 0.12 | 从完全不透明平滑过渡到极淡透明 |
| 相关节点颜色 | 默认色 → 亮蓝色 | RGB 颜色插值，渐变到高亮状态 |
| 相关连线颜色 | 灰色 → 亮蓝色 | RGB 颜色插值，渐变到高亮状态 |
| 箭头颜色 | 与连线同步 | 使用相同的颜色插值 |
| 标签边框/文字 | 灰色 → 蓝色 | 渐变到高亮状态 |

**技术实现**：
- 使用 `requestAnimationFrame` 实现帧动画
- 使用 `useRef` 存储动画状态，避免 React 重渲染性能问题
- 颜色插值函数：`lerpColor(color1, color2, t)` 在 RGB 通道间线性插值
- 通过 `d3ReheatSimulation()` 触发 ForceGraph2D 持续重绘

**交互体验**：
- 鼠标悬浮节点时，无关节点和连线丝滑淡出
- 相关节点和连线颜色从默认色丝滑渐变到亮蓝色
- 鼠标离开节点时，所有元素平滑恢复原状
- 动画过程流畅，视觉过渡自然不突兀

### 5.5 渐变显示效果

```
缩放级别变化时：
- 缩小视图（globalScale 降低）：连线始终显示，箭头、关系标签、节点标签平滑淡出
- 默认视图（globalScale 约 0.8-1.0）：连线、箭头、关系标签完全显示，节点标签渐变显示
- 放大视图（globalScale > 1.0）：连线、箭头、关系标签、节点标签完全显示

悬浮或拖动节点时：
- 焦点节点：亮蓝色 #4dabf7，节点变大 0.5px
- 相关节点：深蓝 #2e86de
- 相关连线：亮蓝色 #4dabf7
- 默认连线：灰色
- 无关节点和连线：极淡蓝灰，透明度 25%，几乎融入背景
```

### 5.6 空状态

- 无知识时显示"暂无知识数据"
- 有知识但无关系时仍显示图谱（只有节点，无边）

## 六、技术实现

### 6.4 核心实现

**节点绘制**：Canvas 2D API
- 圆形路径 + 颜色填充
- 阴影效果增加质感
- 悬浮时节点变大 0.5px（不改变颜色）
- 透明度渐变控制（无关节点 0.45）

**连线绘制**：Canvas 2D API
- 从节点边缘开始绘制
- 连线紧贴变大后的节点边缘
- V形箭头（两条线段）
- 关系标签背景圆角矩形
- 透明度渐变控制（无关连线 0.45）

**标签绘制**：
- 位置固定基于基础节点大小，不随悬浮变大而移动
- 与节点边缘间距 2px

**力导向配置**：
- forceX / forceY：向心力（使用 d3-force 实现，center force 无 strength 方法）
- linkForce.strength：连线吸引力
- chargeForce.strength：节点互斥力

### 6.5 自定义鼠标碰撞检测（v5.4）

**问题背景**：
- react-force-graph-2d 默认使用 `nodeRelSize` 作为碰撞检测半径
- 实际绘制节点时会根据出度动态放大：`sizeMultiplier = 1 + Math.sqrt(outDegree) * 0.3`
- 导致碰撞检测范围小于视觉范围，用户需要移动到节点中心附近才触发高亮

**解决方案**：
禁用 react-force-graph 的默认悬浮检测，使用 Canvas 原生鼠标事件实现精确碰撞检测。

**技术实现**：

```typescript
// 1. 通过 DOM 查询获取 canvas 元素
const canvas = document.querySelector('.force-graph-container canvas')

// 2. 坐标转换：屏幕坐标 → 图形坐标
const screenToGraphCoords = (screenX: number, screenY: number) => {
  const zoom = graphRef.current.zoom()
  const center = graphRef.current.centerAt()
  const rect = canvas.getBoundingClientRect()

  const graphX = (screenX - rect.left - rect.width / 2) / zoom + center.x
  const graphY = (screenY - rect.top - rect.height / 2) / zoom + center.y

  return { graphX, graphY }
}

// 3. 精确碰撞检测：使用视觉半径
const findNodeAtPosition = (graphX: number, graphY: number) => {
  for (const node of nodes) {
    // 计算视觉半径（与绘制函数一致）
    const sizeMultiplier = Math.min(1 + Math.sqrt(node.outDegree) * 0.3, 3)
    const visualRadius = config.nodeSize * sizeMultiplier

    // 计算鼠标到节点中心的距离
    const dx = graphX - node.x
    const dy = graphY - node.y
    const distance = Math.sqrt(dx * dx + dy * dy)

    if (distance <= visualRadius) {
      return node
    }
  }
  return null
}

// 4. 绑定事件
canvas.addEventListener('mousemove', handleMouseMove)
canvas.addEventListener('mouseleave', handleMouseLeave)
canvas.addEventListener('click', handleClick)
```

**注意事项**：
- 使用 `useRef` 存储最新的 `graphData`、`config`、`handleNodeClick`，避免 useEffect 依赖问题
- 事件监听器需要在组件卸载时清理
- 坐标转换需考虑缩放级别（zoom）和平移偏移（centerAt）
