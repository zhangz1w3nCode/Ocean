# 工作流应用功能设计规范

> 本文档描述工作流"应用"功能的设计规范，允许将工作流应用到其他业务模块（如命令、技能）。

---

## 一、功能概述

### 1.1 功能定位

工作流应用功能允许用户将当前工作流引用并创建为其他业务模块的实体，实现工作流的复用和联动。

### 1.2 应用场景

- 将工作流转换为可执行的命令
- 将工作流转换为可复用的技能
- 在命令/技能中引用工作流作为执行依据
- 实现工作流与命令、技能的联动

## 三、应用弹窗设计（两步交互）

### 3.1 交互流程

采用两步交互设计，首先选择目标模块，然后编辑具体内容：

1. **第一步 - 选择模块**：用户选择要应用到的目标（命令/技能）
2. **第二步 - 编辑内容**：输入名称并编辑内容

### 3.4 应用目标选项

| 目标 | 图标 | 描述 | 颜色 |
|------|------|------|------|
| 命令 | Terminal | 创建一个命令，引用当前工作流 | bg-gray-100 text-gray-600 |
| 技能 | Wand2 | 创建一个技能，引用当前工作流 | bg-violet-100 text-violet-600 |

### 3.5 名称输入

- **命令默认值**：`execute-{工作流名称}-workflow`
- **技能默认值**：`{工作流名称}-workflow-skill`
- **格式**：小写字母、数字、连字符
- **唯一性**：检查名称是否已存在，已存在则提示用户修改
- **编辑**：用户可自定义名称

### 3.6 技能描述（仅技能时显示）

- **默认值**：空
- **必填字段**：是
- **placeholder**：`可选，描述该技能的用途...`
- **显示条件**：仅在选中技能时显示

### 3.7 表单验证

| 目标 | 字段 | 验证规则 | 提示信息 |
|------|------|----------|----------|
| 命令 | 名称 | 必填 | 请输入命令名称 |
| 命令 | 名称 | 唯一性 | 当前命令已应用 |
| 命令 | 内容 | 必填 | 请输入命令内容 |
| 技能 | 名称 | 必填 | 请输入技能名称 |
| 技能 | 名称 | 唯一性 | 当前技能已应用 |
| 技能 | 描述 | 必填 | 请输入技能描述 |
| 技能 | 内容 | 必填 | 请输入技能内容 |

验证失败时使用 Toast 提示，不关闭弹窗。

### 3.7 内容编辑器

采用编辑/预览双模式设计：

编辑模式：使用 MarkdownEditor 组件
- 支持 Markdown 语法高亮
- 支持 @ 引用功能
- 8 行高度

预览模式：使用 MarkdownRenderer 组件
- 渲染最终效果
- 最大高度 300px，超出滚动

### 3.7 内容模板

```markdown
# 用户需求
- $ARGUMENTS

# 工作流
- 位置：`.claude/workflows/{workflowName}/WORKFLOW.md`

# 强制要求
- 强制按照`工作流`的内容执行

# 禁止事项
- 不按照强制按照`工作流`的内容执行 注意替换内容中的工作流位置
```

## 四、创建逻辑

### 4.1 创建流程

1. 用户点击"应用"按钮
2. 检查工作流是否已保存
   - 未保存：触发保存操作
   - 已保存：打开应用弹窗（第一步）
3. 用户选择目标模块（命令/技能）
4. 进入编辑步骤，显示名称输入框和内容编辑器
5. 用户点击确认（创建命令/创建技能）
6. 验证表单字段（名称、描述、内容）
   - 验证失败：显示 Toast 提示，不关闭弹窗
7. 检查名称唯一性
   - 名称已存在：显示 Toast 提示，不关闭弹窗
8. 创建目标文件
9. 关闭弹窗，显示成功提示

### 4.2 生成的命令结构

```typescript
interface CommandFile {
  id: string              // cmd-{timestamp}
  name: string            // execute-{workflow-name}-workflow
  description: string     // 执行 "{workflowName}" 工作流的命令
  content: string         // 命令内容（用户可编辑）
  type: 'command'
  createdAt: string       // ISO时间戳
  updatedAt: string       // ISO时间戳
}
```

存储位置：`.claude/commands/{name}.md`

### 4.3 生成的技能结构

技能采用目录结构存储，创建时会生成以下文件：

```
.claude/skills/{skill-name}/
├── SKILL.md              # 主技能文件
```

SKILL.md 文件格式：

```markdown
---
name: {skill-name}
description: 执行 "{workflowName}" 工作流的技能
---

# 用户需求
- $ARGUMENTS

# 工作流
- 位置：`.claude/workflows/{workflowName}/WORKFLOW.md`

# 强制要求
- 强制按照`工作流`的内容执行

# 禁止事项
- 不按照强制按照`工作流`的内容执行 注意替换内容中的工作流位置
```

## 六、状态管理

### 6.1 ApplyModal 状态

```typescript
interface ApplyModalState {
  step: 'select' | 'edit'                     // 当前步骤
  selectedTarget: 'command' | 'skill' | null  // 选中的应用目标
  viewMode: 'edit' | 'preview'                // 编辑/预览模式
  commandName: string                         // 命令名称（可编辑）
  commandContent: string                      // 命令内容（可编辑）
  skillName: string                           // 技能名称（可编辑）
  skillDescription: string                    // 技能描述（可编辑）
  skillContent: string                        // 技能内容（可编辑）
  isCreating: boolean                         // 创建中状态
}
```

### 6.2 依赖的 Store

- `useWorkflowStore` - 获取工作流信息（名称、hasMetadata）
- `useCommandStore` - 创建命令（addCommandFile）
- `useSkillStore` - 创建技能（createSkill）
- `useToastStore` - 显示操作提示

## 七、设计要点

### 7.1 视觉规范

- 命令按钮颜色：浅灰色系（bg-gray-100）
- 技能按钮颜色：紫罗兰色系（bg-violet-100）
- 选择步骤弹窗：较小尺寸（lg）
- 编辑步骤弹窗：较大尺寸（xl）
- 编辑器：编辑模式使用 MarkdownEditor，预览模式使用灰色背景卡片

### 7.2 交互规范

- 两步交互：先选择模块，再编辑内容
- 点击选项直接进入编辑步骤
- 编辑步骤可通过"返回"按钮回到选择步骤
- 默认显示预览模式，减少用户认知负担
- 名称可编辑，但提供合理的默认值
- 内容可编辑，满足用户自定义需求
- 确认按钮文字根据选择的目标动态变化（"创建命令" / "创建技能"）

### 7.3 错误处理

- 工作流ID无效：显示错误提示
- 名称重复：显示警告提示，要求用户修改名称
- 内容为空：显示警告提示
- 创建失败：显示错误提示，保留弹窗状态
